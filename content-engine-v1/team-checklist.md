# Team Checklist — AI Content Engine V1

> เอกสารนี้รวม Checklist สำหรับแต่ละบทบาทในทีม ทุก Checklist ต้องทำจริงก่อนดำเนินการต่อ
>
> **วิธีใช้:** Print หรือ Copy ไปใช้งานจริง ทำ Check (✅) ทุกรายการก่อนส่งงานต่อ

---

## บทบาทและความรับผิดชอบ

| บทบาท | ผู้รับผิดชอบ | งานหลัก |
|-------|-----------|--------|
| Owner/ผู้บริหาร | เจ้าของธุรกิจ | อนุมัติ Direction + Final Review |
| Content Strategist | ผู้วางแผนคอนเทนต์ | Ideas + Briefs + Performance |
| Script Reviewer | ผู้ตรวจ Script | Compliance + Quality |
| Designer/Editor | ช่างภาพ/ตัดต่อ | Production + Visual |
| Social Media Scheduler | ผู้จัดการ Social | Schedule + Publish |
| Performance Reviewer | ผู้วิเคราะห์ผล | KPI + Reporting |

---

## 1. Owner / ผู้บริหาร

### รายสัปดาห์ (ทุกวันจันทร์)
- [ ] อ่าน Weekly Review Report จาก Performance Reviewer
- [ ] Review Content Score ของทุกแบรนด์
- [ ] Approve/Reject ทิศทาง Content สัปดาห์ถัดไป
- [ ] ยืนยัน Priority ของแต่ละแบรนด์
- [ ] ตรวจสอบ Budget สำหรับ Paid Ads ถ้ามี

### ก่อน Publish (ทุกชิ้น — สำหรับ High-priority Content)
- [ ] อ่าน Script สุดท้ายที่ผ่าน Review แล้ว
- [ ] ดูไฟล์ Video/Image จริง ไม่ใช่แค่ตัวอย่าง
- [ ] ตรวจ Caption และ CTA
- [ ] ยืนยัน Publish Date/Time
- [ ] Sign off ใน Google Sheets (ใส่ชื่อ + วันที่ ใน "Approved By" column)

### ก่อน Publish Promotion/Ads (ทุกชิ้น)
- [ ] ยืนยันราคาและโปรโมชั่นเป็นปัจจุบัน
- [ ] Approve Budget สำหรับ Paid
- [ ] ตรวจ Compliance ตาม Approval Gate 3 และ 6
- [ ] Sign off เป็นลายลักษณ์อักษร

### รายเดือน
- [ ] Review ภาพรวม KPI ทุกแบรนด์
- [ ] ตัดสินใจ Scale/Pause แต่ละแบรนด์
- [ ] Review Budget ที่ใช้ไป
- [ ] Update Brand Direction ถ้ามีการเปลี่ยนแปลง

## 2. Content Strategist

### เมื่อเริ่มสัปดาห์ใหม่ (วันจันทร์)
- [ ] อ่าน Weekly Review Report
- [ ] ดู Top Performers และ Failed Content
- [ ] อัปเดต Idea Bank จากผล Review
- [ ] วางแผน Content Calendar สัปดาห์นี้
- [ ] ยืนยัน Capacity ของ Writer และ Editor

### เมื่อสร้าง Idea
- [ ] ใช้ Prompt 01 สร้าง Idea 10 ชิ้น
- [ ] ตรวจทุก Idea ว่ามี Hook ชัดเจน
- [ ] ให้ Score แต่ละ Idea (1-10)
- [ ] ตรวจ Risk Note ทุก Idea
- [ ] เลือก Top 3-5 Idea ที่ Score ≥ 6
- [ ] กรอก Idea Bank ใน Google Sheets ครบทุกคอลัมน์
- [ ] เปลี่ยน Status เป็น "Selected"
- [ ] ส่ง Owner Approve (สำหรับ High Risk Idea)

### เมื่อ Assign Script Writer
- [ ] เปลี่ยน Status เป็น "Approved" หลัง Owner Sign off
- [ ] สร้าง Script ID ใน Script Queue
- [ ] Assign Writer ที่ชัดเจน + Deadline
- [ ] ส่ง Writer: Idea, Brand Playbook, Prompt 02
- [ ] บันทึก Assign ใน Google Sheets

### เมื่อสร้าง Creative Brief
- [ ] Script ผ่าน Review แล้ว (Script Status = "Approved")
- [ ] ใช้ Prompt 04 สร้าง Brief
- [ ] ตรวจ Brief ครบทุก Section: Shot List, Visual, Text Overlay, Thumbnail, Assets
- [ ] สร้าง Brief ID ใน Creative Briefs Sheet
- [ ] Assign Designer/Editor + Deadline
- [ ] ส่ง Brief พร้อม Assets ทั้งหมด
- [ ] บันทึก Brief Status เป็น "Sent to Editor"

### เมื่อ Track Performance
- [ ] ดู Published Content ทุก 48 ชั่วโมง
- [ ] Note Engagement ผิดปกติ (ดีหรือแย่กว่าคาดมาก)
- [ ] Coordinate กับ Performance Reviewer ในวัน Weekly Review
- [ ] อัปเดต Repurpose Decision ใน KPI Tracker

## 3. Script Reviewer

### เมื่อได้รับ Script ให้ Review
- [ ] ตรวจว่า Script มีครบทุก Section: Hook, Setup, Body, CTA, B-roll, Text Overlay
- [ ] อ่าน Script ด้วยมุมมองของผู้ชม: "ถ้าฉันดูวิดีโอนี้ ฉันจะหยุดดูไหม?"
- [ ] ตรวจ Compliance ตาม Brand Playbook Do/Don't

### Compliance Checklist (ทำทุก Script)
- [ ] ไม่มี Medical Claims ที่ไม่มีหลักฐาน (Harmony Sauna, ธรรมะดีดี)
- [ ] ไม่มี Financial/ROI Guarantee (AMP Pattaya, Affiliate)
- [ ] ไม่มีราคาจริงหรือ Transit Time Guarantee (The World Freight)
- [ ] ไม่มี Get-rich-quick Language (Affiliate/AI)
- [ ] Testimonial มีการขออนุญาตแล้ว (ถ้ามี)
- [ ] CTA ถูกต้องและทำได้จริง
- [ ] Tone ตรงกับ Brand Playbook

### Quality Checklist (ทำทุก Script)
- [ ] Hook ดึงดูดใน 3 วินาที
- [ ] Setup ชัดเจนว่าวิดีโอนี้เกี่ยวกับอะไร
- [ ] Body มี Pattern Interrupt อย่างน้อย 1 จุด
- [ ] CTA ชัดเจน ไม่เกิน 1 CTA
- [ ] ความยาวเหมาะกับ Platform ที่ระบุ

### หลัง Review
- [ ] อัปเดต Script Status ใน Script Queue
- [ ] ถ้า Approve: บันทึก "Approved By" + วันที่
- [ ] ถ้า Reject: กรอก Revision Notes ชัดเจน บอกว่าต้องแก้อะไร
- [ ] แจ้ง Writer และ Strategist ทันที
- [ ] ทำให้เสร็จภายใน 48 ชั่วโมงหลังได้รับ Script

## 4. Designer / Video Editor

### เมื่อได้รับ Creative Brief
- [ ] อ่าน Brief ทั้งหมดก่อนเริ่มงาน
- [ ] ตรวจว่า Assets ทั้งหมดอยู่ใน Google Drive ครบแล้ว
- [ ] ถ้า Assets ไม่ครบ: แจ้ง Strategist ทันที อย่าเริ่มงาน
- [ ] ยืนยัน Deadline กับ Strategist
- [ ] ยืนยัน Export Format และ File Naming ที่ถูกต้อง

### ระหว่าง Production
- [ ] ทำตาม Shot List ในลำดับที่กำหนด
- [ ] ใช้ Color Palette ตาม Brief
- [ ] ตรวจ Text Overlay ทุกจุดว่าถูกต้อง ไม่สะกดผิด
- [ ] ตรวจ Audio Level: Voiceover ชัด, BGM ไม่ดัง
- [ ] ตรวจ Copyright ของ Music และ Footage ทุกอย่าง
- [ ] ใส่ Logo/Brand Elements ตาม Brand Playbook

### ก่อนส่ง Final Review
- [ ] ดูไฟล์ทั้งหมดตั้งแต่ต้นจนจบอย่างน้อย 1 ครั้ง
- [ ] ตรวจ Video Resolution (ต้องไม่พร่ามัว)
- [ ] ตรวจ Audio (ไม่มีเสียงแตก หรือ Pop)
- [ ] ตรวจความยาวว่าตรงกับ Brief ± 3 วินาที
- [ ] Export ด้วย Spec ที่กำหนดใน Brief
- [ ] ตั้งชื่อไฟล์ตาม Convention: `[BrandCode]-[ContentID]_v[Version]_[Platform]_[Date].mp4`
- [ ] Upload ขึ้น Google Drive folder ที่กำหนด
- [ ] แจ้ง Strategist ว่างานเสร็จ + อัปเดต Brief Status เป็น "Review Ready"

### File Naming Convention
```
HS-001_v1_TikTok_20260510.mp4       ← Harmony Sauna
AMP-002_v1_Facebook_20260510.mp4    ← AMP Pattaya
TWF-003_v2_YouTube_20260510.mp4     ← The World Freight
DDD-004_v1_YouTube_20260510.mp4     ← ธรรมะดีดี
AIA-005_v1_TikTok_20260510.mp4      ← Affiliate/AI Automation
```

## 5. Social Media Scheduler

### วันก่อน Publish
- [ ] ตรวจ Content Calendar ว่ามีคอนเทนต์ Ready สำหรับพรุ่งนี้
- [ ] ดาวน์โหลดไฟล์จาก Google Drive
- [ ] ตรวจ Caption ครบถ้วน: Hashtags, CTA, Link
- [ ] ตรวจ Caption ความยาวเหมาะกับ Platform
- [ ] ยืนยัน Account ที่จะ Publish (ไม่ Post ผิด Account)
- [ ] ตรวจ Status ใน Google Sheets ว่า "Final Review" ผ่านแล้ว

### วัน Publish
- [ ] Post ตรงเวลาตาม Calendar (± 15 นาที)
- [ ] Copy-paste Caption จาก Google Sheets อย่างระมัดระวัง
- [ ] ตรวจ Post หลัง Publish ว่า Caption ครบ, ไม่มีข้อผิดพลาด
- [ ] บันทึก Published URL ใน Publishing Calendar
- [ ] เปลี่ยน Status เป็น "Published"
- [ ] แจ้ง Team ใน Group Chat ว่า Post แล้ว

### หลัง Publish (24 ชั่วโมงแรก)
- [ ] ตรวจ Comment ทุก 4 ชั่วโมง (ช่วงเวลาทำงาน)
- [ ] Reply Comment ที่ถามคำถามภายใน 2 ชั่วโมง
- [ ] Note Comment ผิดปกติหรือ Negative → แจ้ง Strategist
- [ ] จดบันทึก Engagement เบื้องต้น (Views, Likes หลัง 24 ชั่วโมง)
- [ ] ส่งยอดเบื้องต้นให้ Performance Reviewer

### สิ่งที่ต้องทำเมื่อเกิดปัญหา
- มีข้อมูลผิดพลาดใน Post → แจ้ง Strategist ทันที → Delete ถ้าจำเป็น
- มี Negative Comment ที่เป็น Crisis → แจ้ง Owner ทันที ห้าม Reply เอง
- Account ถูก Ban/Restrict → แจ้ง Owner และ Strategist ทันที

## 6. Performance Reviewer

### หลัง Publish 48 ชั่วโมง
- [ ] เข้า Platform Analytics ของทุกคอนเทนต์ที่ Publish
- [ ] กรอก KPI ทั้งหมดใน KPI Tracker Sheet
- [ ] กรอก Conversion Signal ตามคู่มือใน kpi-tracking.md
- [ ] คำนวณ Content Score ด้วยสูตรใน kpi-tracking.md
- [ ] อัปเดต Status เป็น "Analyzed"

### วัน Weekly Review (ทุกวันจันทร์)
- [ ] รวบรวม KPI ทุกคอนเทนต์ที่ Publish ในสัปดาห์ที่ผ่านมา
- [ ] Note External Factors พิเศษ (วันหยุด, Event, ฯลฯ)
- [ ] ใช้ Prompt 05 วิเคราะห์ผลสำหรับแต่ละแบรนด์
- [ ] กรอกผลใน Weekly Review Sheet ครบทุกคอลัมน์
- [ ] สรุป Top Performers, Failed, Patterns ชัดเจน
- [ ] ส่ง Summary Report ให้ Owner และ Strategist ภายใน 12:00 วันจันทร์

### สิ่งที่ต้องส่งใน Weekly Report
- [ ] คะแนนเฉลี่ยของสัปดาห์ แยกตามแบรนด์
- [ ] Top 3 Content (ระบุ Content ID + คะแนน + เหตุผล)
- [ ] Bottom 3 Content (ระบุ Content ID + คะแนน + เหตุผล)
- [ ] Patterns ที่พบ (อย่างน้อย 3 Patterns)
- [ ] Recommendations สำหรับสัปดาห์หน้า
- [ ] สิ่งที่ควรทดลอง (A/B Test)

## Escalation Guide — เมื่อมีปัญหา

| สถานการณ์ | ผู้รับแจ้ง | เวลาที่ต้องแจ้ง |
|---------|----------|--------------|
| Content ผิดพลาดหลัง Publish | Strategist + Owner | ทันที |
| Comment Negative ที่เป็น Crisis | Owner | ทันที |
| Script มี High-risk Claim | Reviewer + Owner | ทันที |
| Production ล่าช้าเกิน Deadline | Strategist | 24 ชั่วโมงก่อน Deadline |
| Account ถูก Ban/Restrict | Owner | ทันที |
| KPI ต่ำกว่า Benchmark 3 สัปดาห์ติดกัน | Owner + Strategist | ใน Weekly Review |
